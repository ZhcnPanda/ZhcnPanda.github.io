· 前言

  突然有一天晚上突发奇想，想做一个博客，记录一下踩坑日常，虽然之前也弄过，不过基本上都是半途而废。考虑到经济实惠和稳定性，选择Github Page托管，虽然有的时候抽风，不过问题不大。至于博客系统选择的话，找了很久，发现Hugo最适合我了，主要是简洁，够用，因为是托管在Github Page上面，由于是国外网站，花里胡哨的功能加了太多，访问速度也会下降很多。
  文章Hugo部署部分使用Windows平台示例，其他平台可参考Hugo官方文档。

· 在本地部署Hugo

  • Hugo官方提供了三种安装方式

    • 直接下载编译好的二进制程序
    • 使用包管理器安装，例如Chocolatey，Scoop，Winget
    • 自己从源代码构建

  • 最简单方便的方法就是第一个，下载运行，一气呵成，文章使用第一种方法，其他安装方式见 [前言] 部分的Hugo官方文档。

  1. 前往Hugo的Github Release，根据你的设备架构选择对应的程序下载，获取Hugo最新版本。我这边选择 hugo_extended_0.123.3_windows-amd64.zip

    • 因为兼容性的问题，在Windows平台上，大部分Hugo的第三方主题都需要使用扩展版本，所以这里我们下载带有 “extended” 标识的版本）。

  2. 解压缩包后文件结构如下:

- hugo_extended_0.123.3_windows-amd64
    │
    ├─── hugo.exe
    │
    ├─── LICENSE
    │
    └─── README.md
  1. 将此目录添加到 PATH 环境变量:
    Windows设置 > 系统 > 关于 > 高级系统设置 > 环境变量 > 选择'Path' > 编辑 > 浏览 > 选择此目录 > 保存

    打开Windows PowerShell,输入以下命令检查Hugo是否部署完毕：
  • Hugo官方要求使用PoweShell而不是Windows PowerShell,但是经过这几天我的使用，并没有发现Windows PowerShell存在Bug，如果后续在使用的时候遇到奇怪的报错，尝试使用PowerShell，详请点我。
>  hugo version

  输出版本详细信息即为部署成功。

· 创建并启动Hugo网站

  1. 进入工作目录，输入以下命令创建一个 Hugo 站点:
>  hugo new site <网站名称>
  1. Hugo 必须一同安装主题，这里用我使用的主题MemE,按照MemE官方文档指引，推荐使用Git拉取源码的方式安装主题，这样有利于后续MemE主题的更新,进入到刚刚创建好的站点根目录下，输入以下命令拉取主题源码:
>  git init
>  git submodule add --depth 1 https://github.com/reuixiy/hugo-theme-meme.git themes/meme
  1. 替换 Hugo 默认配置: 在站点根目录下，将 Hugo 默认生成的 hugo.toml 删除，再进入 themes/meme/config-examples/zh-cn/ 目录，将 MemE 提供的 config.toml 复制到站点根目录下。
  2. 使用以下命令，尝试启动 Hugo 服务端:
>  hugo server -D
  1. 浏览器尝试访问: localhost:1313,如果出现首页，说明站点创建成功。(如果无法访问，请先检查是不是默认的1313端口被占用，导致Hugo使用了其他端口，具体查看命令行输出。)

· 发布第一篇文章

  1. 打开站点根目录，可以发现 content 文件夹，这个文件夹就是文章存放目录，包括自定义的Markdown格式展示页面（如：首页，关于等等），由于 Hugo 只识别带有 Front Matter 的 Markdown格式 文章，使用命令可以直接生成带有此标识的文件，所以推荐使用命令行创建空文章。
  2. 发布文章请先关闭 Hugo 服务端: 在命令行界面，直接按下 Ctrl + C 组合键。
  3. 回到站点根目录，输入以下命令创建一个自带 Front Matter 的Markdown格式 文章:
>  hugo new "posts/hello-world.md"
  1. 此时站点根目录 contents 目录下出现了posts文件夹，里面就是刚刚创建的 hello-world.md 文章。

· 站点的完善与个性化

  • 本文章只涉及了最基本的个性化内容，高级玩法移步 MemE主题官方文档。
  • MemE 的大部分站点配置都在根目录的 config.toml 中，并且该文件中贴心的使用的相当多的中文注释，小白稍微研究一下就能看得懂。
  • 对比站点目录和 MemE 主题的目录，细心的网友就发现了：站点根目录下的文件夹和 MemE 主题的目录文件夹基本重合，只不过站点根目录的文件夹大部分是空的。这是因为 Hugo 站点配置文件与个性化资源文件会先从站点根目录下拿取，如果文件不存在，Hugo 就会从 MemE 主题内拿取，因此，如果我们想更改某个资源文件，请务必先把主题内的资源文件复制到 Hugo 站点根目录下对应的文件夹，在进行修改，否则将会严重的影响MemE主题的后续更新（Git同步冲突）。
  1. 修改站点图标。（该操作在站点配置文件中 “网页应用” 部分也有说明）
    1. 将 MemE 主题目录下的 static 文件夹复制到站点根目录下，覆盖站点的 static 文件夹。
    2. 前往 https://realfavicongenerator.net/ 生成相关图标和文件，下载后解压，仅保留 android-chrome-512x512.png、apple-touch-icon.png、mstile-150x150.png、safari-pinned-tab.svg、favicon.ico、site.webmanifest 这6个文件，删除其余文件。
    3. 对照站点根目录下 static 文件夹中对应的文件，进行替换即可。
